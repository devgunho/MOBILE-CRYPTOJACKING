eyezware: Home



   Welcome to eyezware. I host custom, bash, shell scripts, all different; Some just for fun, some more useful. Each script has its own project page and comment system, which I hope you'll use for useful feedback. My old site was eyez-Observe, which was also hosted by Daniel, at zj6dtirxj543frww.onion. Although I do still have the site backed up, I thought it was a good time to rebuild and get my own permanent URL RSA, so that I can keep the same URL if anything should happen to this site. I also wanted a custom name showing in the URL. I've kept the same color scheme, but there is less content than there was at the old URL. That is good though, as I can concentrate more on bash scripting, instead of spreading myself so thin. Also, since I made AViator (now a deb file), I've seen little need to keep maintaining a live playlist, since you can get your own playlists with ease from any paste site. Also there are channel functions to AViator now too, so it is simple to keep the ones you like, even if you only have the web URL, as Streamlink and YouTube-DL take care of the dynamic media URL problem. You can't save a media URL, if it isn't permanent and most aren't, but the web URL that indexes the media file usually stays the same, giving us something to add to our channels.

   My last site had AutoConnect and some very similar scripts within it. If you are looking for it, I am just posting those under the name Vanish now. I will, simply, post notes on dealing with static connections and apps that always get your new IP, in comments within the main script, Vanish. There are other scripts I host here that Vanish calls on, like SUcmds and VPNexclude and I also host usable lists for those scripts to use. In case you are wondering, those lists are for grabbing random information and using it to repopulate certain fields in conf files, every time certain scripts are run. Vanish, itself, is for getting a random VPN connection (you must acquire your own VPN accounts/VPS). It also listens in real time for any connection changes and immediately kills all networking, then reconnects to a new VPN server, with all new random info, including MAC address. The MAC is completely random at default. You will have to change the method a bit to emulate actual hardware IDs. This falls in compliance with, what I call, the FU method. I suggest you check the project page, for more info, if your interested. For reference, the FU method of anonymity does not clone you, but instead gives you a new uniqueness, each time. Please let me know of a web broswer project, if they have similar goals, as I would like to include one into my FU method of connecting. At the moment, FireFox with certain plugins is the best I can suggest right now to randomize settings.

   Not every script is so practical though. I also have a few things for fun, like different types of animated wallpaper, but even this can have a practical purpose. Click a button and have your (favorite?) webcam showing live images as your wallpaper in XFCE4. A few simple tweaks can have it running in other environments too. Maybe you are always checking the terminal for output. It is nice to see system messages or netstat output or any other command you are always running as the desktop background. Maybe you always wanted an animated gif as wallpaper, but gif doesn't animate as a desktop background. If you "explode" the gif with a simple, single command, you can essentially have, the gif as wallpaper. I mean it. It is wallpaper. Not like the numerous things I tried that claimed to be wallpaper, but in reality, just showed as a fullscreen picture, covering my desktop entirely, though some are known to work in other environments. I had no choice, but to create my own for them to work as I had anticipated.

   Even scipts that look easy to dismiss were made for a purpose, like Screenshot-XFCE4. Sure there are tons of screenshot apps, but this one does something special and works well with the live webcam wallpaper scripts, for instance. If you use certain "wp" scripts from this site, you may want to check it out, as it copies the background source picture at its original resolution, rather than snapping an actual screenshot at your navive one. Even the backup scripts were made for a reason. I couldn't find an all in one solution for backing up everything. GUI apps have issues dealing with root files, no backup app had support for all my online storage (which most goofballs call cloud now), none gave easy access to individual files and some even changed my file permissions. Once started, it's automatic, till finished. You will have to edit it to match your file locations, initially, keeping diskspace in mind and dealing with root files means a small configuration to sudoers; small, but not taken lightly, but of course, root files are optional and a different script is called on withink bk-all,which is easy to uncomment, called bk-core. Find it in the left menu if it interests you.

Other information can help a lot too. I hope you like smart ass comments because I intend to fill HashCrack with them. They help to make a completely improbable task a bit more interesting.
